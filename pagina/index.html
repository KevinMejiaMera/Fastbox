Objetivo General

Desarrollar un sistema POS para restaurantes de comida rápida que permita registrar pedidos, procesar ventas, gestionar inventario, administrar usuarios y generar reportes.

Objetivos Específicos

  • Módulo de ventas con interfaz rápida (Django Templates + JS)
  • Inventario con Redis para caché de stock y alertas
  • Usuarios con roles (admin, cajero, cocina)
  • Dashboard de reportes con indicadores clave
  • Calidad basada en ISO 25010, 12207, 9001, 15504, CMMI, OWASP, WCAG

Alcance

Sistema orientado a restaurantes de comida rápida. Cubre el ciclo completo: pedidos, pagos, ticket, inventario y reportes. No incluye facturación electrónica ni integración con proveedores en v1.

Flujo de Venta

  1. Cajero ingresa (sesión Django + Redis)

  6. Movimiento guardado en movimientos_inventario
  7. Ticket generado e impreso

Normativas

ISO 25010 — Calidad

Pruebas funcionales por sprint, monitoreo de tiempos de respuesta, cobertura de código. Evalúa funcionalidad, confiabilidad, usabilidad y mantenibilidad del POS.

ISO 12207 — Ciclo de Vida

Procesos de desarrollo adaptados a Scrum: análisis, diseño, implementación, pruebas y despliegue iterativo en cada sprint.

ISO 9001 — Gestión Calidad

Enfoque al cliente (historias de usuario), mejora continua (retrospectivas), decisiones basadas en datos (métricas) y procesos documentados.

ISO 15504 (SPICE)

Evaluación de capacidad de procesos: desde ejecución básica hasta gestión y medición de resultados en cada sprint del proyecto.

CMMI — Madurez

Prácticas del Nivel 2 (Gestionado): planificación, gestión de requisitos, monitoreo y control, aseguramiento de calidad y configuración con Git.

OWASP — Seguridad

CSRF Django, validación de entradas, bcrypt/PBKDF2, control por roles y sesiones gestionadas con Redis.

WCAG — Accesibilidad

Contraste adecuado, navegación por teclado, etiquetas semánticas y diseño responsive para tablets y monitores táctiles.

Conclusiones

Resultados

Sistema POS funcional con 4 módulos completados. Tasa de éxito del 98.5% en transacciones. Todos los sprints entregados a tiempo.

Lecciones

  • Scrum permitió adaptarse a cambios sin afectar el cronograma.
  • Las normativas ISO guiaron la calidad del código.
  • ProjectLibre facilitó la planificación y el seguimiento.
  • Pruebas tempranas redujeron defectos críticos.

Recomendaciones

  • Facturación electrónica y reportes fiscales.
  • API REST para integración con apps móviles.
  • Pasarela de pago (PayPal, Stripe).
  • Pruebas de usabilidad con cajeros reales.
